# Deploying Pulse to Vercel

Two paths. Pick one.

---

## Path A — Vercel Web UI (3 minutes, no CLI)

1. Open <https://vercel.com/new>
2. Click **Import Git Repository** → select `udaysai/pulse-fitness`
3. **Configure Project:**
   - Framework Preset: **Next.js** (auto-detected)
   - **Root Directory**: click **Edit** → set to `apps/web` ← important, this is a monorepo
   - Build Command: leave default (`next build`)
   - Output Directory: leave default (`.next`)
   - Install Command: leave default (`npm install`)
4. **Environment Variables** — click "Add" for each:

   | Name | Value (copy from your `.env.local`) |
   |---|---|
   | `NEXT_PUBLIC_SUPABASE_URL` | `https://fvcmhiuwkjowbnwdztbw.supabase.co` |
   | `NEXT_PUBLIC_SUPABASE_ANON_KEY` | (anon JWT from .env.local) |
   | `SUPABASE_SERVICE_ROLE_KEY` | (service role JWT from .env.local) — **must be marked "Sensitive"** |
   | `GEMINI_API_KEY` | (Gemini key from .env.local) — **must be marked "Sensitive"** |
   | `NEXT_PUBLIC_APP_URL` | (will be set to your Vercel URL after first deploy, e.g. `https://pulse-fitness.vercel.app`) |

5. Click **Deploy**. Wait ~90 seconds.
6. Once deployed, **update Supabase auth settings** with your live URL:
   - Open <https://supabase.com/dashboard/project/fvcmhiuwkjowbnwdztbw/auth/url-configuration>
   - Set **Site URL** to your Vercel URL (e.g. `https://pulse-fitness.vercel.app`)
   - Add to **Redirect URLs**: `https://pulse-fitness.vercel.app/auth/callback`
7. **Update Google OAuth client** (Google Cloud Console → Credentials):
   - Add `https://pulse-fitness.vercel.app` to Authorized JavaScript origins
   - Keep the existing `https://fvcmhiuwkjowbnwdztbw.supabase.co/auth/v1/callback` redirect URI (no change needed — Google flow always lands at Supabase first)

Done. Your live URL is now `https://<auto-generated>.vercel.app`.

## Path B — Vercel CLI (if you prefer command line)

```bash
# One-time
vercel login        # opens browser
cd "/Users/supernova/Documents/Health- Superapp/apps/web"
vercel link         # answer prompts: link to "pulse-fitness", root = current dir
```

Then push env vars (run from `apps/web/`):

```bash
vercel env add NEXT_PUBLIC_SUPABASE_URL production
vercel env add NEXT_PUBLIC_SUPABASE_ANON_KEY production
vercel env add SUPABASE_SERVICE_ROLE_KEY production
vercel env add GEMINI_API_KEY production
vercel env add NEXT_PUBLIC_APP_URL production
```

Deploy:

```bash
vercel deploy --prod
```

---

## Continuous deployment (after first deploy)

Every push to `main` on GitHub auto-deploys to production via Vercel's GitHub integration. PRs get preview deploys with their own URL.

To trigger manually: `vercel deploy --prod`.

4. On iPhone Safari → tap Share → **Add to Home Screen** → should install as fullscreen PWA
